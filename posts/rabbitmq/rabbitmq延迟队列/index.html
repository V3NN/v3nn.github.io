RabbitMQ 延迟队列

1、基本介绍

1.1概述

延时队列是一种队列，内部的元素是有序的，元素出队和入队室友方向性的（元素从一端进入、从另一端取出）。跟普通的队列不一样的是，普通队列中的元素总是等着希望被早点取出处理，而延时队列中的元素则是希望被在指定时间得到取出和处理，所以延时队列中的元素是都是带时间属性的，通常来说是需要被处理的消息或者任务。

1.2为什么要使用延时队列

假设有如下场景：

  • 订单在十分钟之内未支付则自动取消
  • 用户发起投诉，如果一天内没有得到处理则通知相关人员
  • 用户注册后，如果三天内未进行登录则进行短信提醒
  • ….

如上这些场景都有一个特点，需要在某个时间发生之后或者发生之前的指定时间完成某一项任务。如订单生成后的十分钟内检查订单支付状态，将未支付的订单进行关闭。

当然除了延时队列，也可以有一些其他办法实现。比如使用定时任务，一直轮询数据，每秒查一次去除需要被处理的数据处理就完事了。

如果数据量比较少，确实可以这样做，但对于数据量比较大并且时效较强的场景，短期内未支付的订单可能会很多，对庞大的数据量仍旧使用轮询的方式是不可取的，很可能在一秒内无法完成所有订单的检查，同时会给数据库带来很大压力，无法满足业务要求而且性能低下。

2、原理

2.1 实现方式

RabbitMQ 实现延迟队列有以下两种实现方式：

  • 通过TTL + 死信队列实现延迟任务

原队列（正常队列）不设置消费者，当消息在原队列中达到 TTL 后，由于还未被消费会被转发到绑定的死信交换器，消费者从死信队列中消费消息，从而实现消息的延迟处理。

需要注意的是，因为队列先进先出的特点， 如果消息延时的时间不同，例如第一条消息延时10S、第二条消息延时5S、第三条消息延时1S，后面的消息需要等待第一条消息（延时10S）消费完才能消费，当第一条消息未被消费则后续的消息都会被阻塞，即使消息设置了更短的延时。

如果一个队列中的消息延迟时间都一致，就可以避免这个问题，因此可以针对不同的延迟时间对应多创建几个队列。

  • 通过官方提供的延迟插件实现延迟功能

通过创建一个延迟交换机（Delay Exchange），延迟消息首先会把消息投递到延迟交换机，并不是直接将消息投递业务队列（所以不会立即执行），而是存储到Mnesia ( Erlang 运行时中自带的一个数据库管理系统），等到消息到达可投递时间才会被投递到目标队列中，从而实现延迟队列。

2.2 优缺点分析

  • 使用死信队列存在时序问题，它不能实现随机延迟任务，每个无消费者的队列上只能设置一个 TTL，所以只能实现固定过期时间的延迟任务。

  • 使用延迟插件则存在消息丢失问题以及高并发问题

    • 消息丢失：消息在真的被投递到目标消息队列之前，是存放在接收到了这个消息的服务端本地的 Mnesia 里面。也就是说，如果这个时候还没有刷新磁盘，那么消息就会丢失；如果这个节点不可用了，那么消息也同样会丢失。
    • 高并发问题：这种实现方式不支持高并发场景，因为它只有一个延迟交换机，当高并发或数据量比较大时执行效率就会比较低。

3、代码示例

以下示例使用Docker运行RabbitMQ，PHP进行代码编写并已安装RabbitMQ相应扩展

3.1 TTL + DLX

编写生产者代码

producer.php

<?php

require_once  '../vendor/autoload.php';

use PhpAmqpLib\Connection\AMQPStreamConnection;
use PhpAmqpLib\Message\AMQPMessage;

$connection = new AMQPStreamConnection('rabbitmq', 5672, 'rabbit', 'rabbit');
$channel = $connection->channel();

// 定义正常交换机和队列（带TTL和DLX）
$delayExchange = 'normal_exchange';
$delayQueue = 'delay_queue';
$dlxExchange = 'dlx_exchange';
$targetQueue = 'target_queue';
// 10秒延迟（单位毫秒）
$delay = 10000;  

// 声明死信交换机
$channel->exchange_declare($dlxExchange, 'direct', false, true, false);

// 声明目标队列
$channel->queue_declare(
    $targetQueue, 
    false, 
    true, 
    false, 
    false
);
$channel->queue_bind($targetQueue, $dlxExchange, 'dlx_routing_key');

// 声明延迟队列（设置TTL和DLX）
$args = new \PhpAmqpLib\Wire\AMQPTable([
    'x-dead-letter-exchange' => $dlxExchange,
    'x-dead-letter-routing-key' => 'dlx_routing_key',
    'x-message-ttl' => $delay,
]);

$channel->exchange_declare($delayExchange, 'direct', false, true, false);
$channel->queue_declare(
    $delayQueue, 
    false, 
    true, 
    false, 
    false,
    false,
    $args
);
$channel->queue_bind($delayQueue, $delayExchange, 'delay_key');

$messageBody = 'Hello Delayed Message!';
// 发送延迟消息
$message = new AMQPMessage($messageBody, [
    'delivery_mode' => AMQPMessage::DELIVERY_MODE_PERSISTENT
]);

$channel->basic_publish($message, $delayExchange, 'delay_key');

echo "Sent {$messageBody} with delay {$delay}ms \n";
$datetime = date('Y/m/d H:i:s');
echo "成功发送延迟消息 : {$messageBody} , {$datetime} \n";

$channel->close();
$connection->close();

编写消费者代码

consume.php

<?php 

require_once  '../vendor/autoload.php';

use PhpAmqpLib\Connection\AMQPStreamConnection;

$connection = new AMQPStreamConnection('rabbitmq', 5672, 'rabbit', 'rabbit');
$channel = $connection->channel();

$targetQueue = 'target_queue';

echo "正在等待延迟队列消息, waiting... \n";
$callback = function ($message) {
    echo $message->body . '-------' . date('Y/m/d H:i:s') . "\n";
    $message->ack();
};

$channel->basic_qos(null, 1, null);
$channel->basic_consume(
    $targetQueue, 
    '', 
    false, 
    false, 
    false, 
    false, 
    $callback
);

while ($channel->is_consuming()) {
    $channel->wait();
}
 
// 关闭连接
$channel->close();
$connection->close();

执行消费者

php consume.php

# 输出如下
正在等待延迟队列消息, waiting...

执行生产者发送消息

php producer.php

# 输出如下
Sent Hello Delayed Message! with delay 10000ms
成功发送延迟消息 : Hello Delayed Message! , 2025/03/26 16:47:32

十秒后会在消费端看到消息被接收

Hello Delayed Message!-------2025/03/26 16:47:42

关键点说明

  1. 队列参数配置

    • x-dead-letter-exchange：指定死信交换机
    • x-dead-letter-routing-key：死信路由键
    • x-message-ttl：队列级别的统一TTL设置

  2. 消息级别TTL（可选）

    可以在发布消息时单独设置TTL

    $message = new AMQPMessage($data, [
    'expiration' => '5000' // 5秒（字符串类型）
]);

  3. 注意事项

    • 队列TTL和消息TTL同时存在时，取较小的值
    • RabbitMQ 只会在消息到达队列头部时检查过期
    • 需要先声明死信交换机和目标队列
    • 消息需要被拒绝或过期才会进入死信队列

3.2 延迟插件

安装对应版本的延迟插件

https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases

  • 进入容器后，下载对应版本插件并启用
# 进入容器
docker exec -ti rabbitmq bash

# rabbitmq插件指定目录（不同版本镜像目录地址不同，更具自己的情况替换）
cd /plugins/

# 下载插件（根据自己rabbitmq版本下载，我这里使用的v3.8.0版本）
wget https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/download/3.8.0/rabbitmq_delayed_message_exchange-3.8.0.ez

# 启用插件
rabbitmq-plugins enable rabbitmq_delayed_message_exchange
  • 重启RabbitMQ服务
docker restart rabbitmq
  • 检查插件是否已经启用
# 进入容器
docker exec -ti rabbitmq bash

rabbitmq-plugins list -e

编写生产者代码

producer.php

<?php 

// 以实际项目路径为准
require_once  '../vendor/autoload.php';

use PhpAmqpLib\Connection\AMQPStreamConnection;
use PhpAmqpLib\Message\AMQPMessage;

// 连接到RabbitMQ服务器
$connection = new AMQPStreamConnection('rabbitmq', 5672, 'rabbit', 'rabbit');
$channel = $connection->channel();
 
// 声明一个具有延迟插件的自定义交换机
$args = new \PhpAmqpLib\Wire\AMQPTable([
    'x-delayed-type' => \PhpAmqpLib\Exchange\AMQPExchangeType::FANOUT
]);
$channel->exchange_declare('delayed_exchange', 'x-delayed-message', false, true, false, false, false, $args);
 
$messageBody = 'Hello RabbitMQ!';
$delay = 5000; // 单位是毫秒
$headers = new \PhpAmqpLib\Wire\AMQPTable(['x-delay' => $delay]);
$message = new AMQPMessage($messageBody, ['delivery_mode' => 2]);
$message->set('application_headers', $headers);
 
// 发布消息到交换机
$channel->basic_publish($message, 'delayed_exchange', 'delayed_key');
 
echo "Sent {$messageBody} with delay {$delay}ms \n";
$datetime = date('Y/m/d H:i:s');
echo "成功发送延迟消息 : {$messageBody} , {$datetime} \n";
 
// 关闭连接
$channel->close();
$connection->close();

?>

编写消费者代码

consume.php

<?php 

// 以实际项目路径为准
require_once  '../vendor/autoload.php';

use PhpAmqpLib\Connection\AMQPStreamConnection;
use PhpAmqpLib\Message\AMQPMessage;
 
// 连接到RabbitMQ服务器
$connection = new AMQPStreamConnection('rabbitmq', 5672, 'rabbit', 'rabbit');
$channel = $connection->channel();
 
// 声明一个具有延迟插件的自定义交换机
$args = new \PhpAmqpLib\Wire\AMQPTable([
    'x-delayed-type' => \PhpAmqpLib\Exchange\AMQPExchangeType::FANOUT
]);
$channel->exchange_declare('delayed_exchange', 'x-delayed-message', false, true, false, false, false, $args);
 
// 声明延迟队列
$channel->queue_declare('delayed_queue', false, true, false, false);
 
// 绑定队列到交换机
$channel->queue_bind('delayed_queue', 'delayed_exchange', 'delayed_key');
 
echo "正在等待延迟队列消息, waiting... \n";
$callback = function (AMQPMessage $message) {
    echo $message->body . '-------' . date('Y/m/d H:i:s') . "\n";
    $message->ack();
};
 
$channel->basic_consume(
    'delayed_queue',
    '',
    false,
    false,
    false,
    false,
    $callback
);
 
while ($channel->is_consuming()) {
    $channel->wait();
}
 
// 关闭连接
$channel->close();
$connection->close();

?>

至此，延迟插件的延时效果已实现。

4、总结

  • RabbitMQ 中可以通过TTL + 死信队列以及官方提供的延迟插件来实现延时任务。

  • TTL + 死信队列存在时序问题，可以针对不同的延迟时间创建多个几个队列来避免。

  • 使用插件可以实现更细粒度的延时控制，但插件的引入可能会带来性能上的开销，并且在高并发场景下，可能会出现消息积压。
